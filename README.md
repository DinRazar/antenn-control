# Antenna Control CLI + Emulator

Небольшой учебный проект для работы с протоколом управления антенной.

В репозитории есть:

- `antenna_ctl.py` — консольный клиент для отправки команд антенне по TCP или serial.
- `antenna_emulator.py` — офлайн-эмулятор антенны для тестирования без реального устройства.

Проект сделан для отладки логики протокола, проверки контрольных сумм и первичной проверки команд до подключения к реальному железу.

## Что умеет

### Клиент `antenna_ctl.py`

- Подключение по TCP.
- Подключение по serial.
- Формирование команд по протоколу антенны.
- Автоматический расчёт checksum.
- Проверка checksum во входящих кадрах.
- Простое консольное меню.
- Режим отправки raw payload.
- Калькулятор контрольной суммы.
- Разбор ответа `show` в человекочитаемом виде.

### Эмулятор `antenna_emulator.py`

- Эмулирует TCP-устройство, которое принимает команды антенны.
- Проверяет checksum входящих кадров.
- Печатает логи `RX / PAY / XOR / TX`.
- Возвращает тестовые ответы на основные команды.
- Позволяет тестировать клиент без доступа к реальной антенне.
- Имеет `demo`-режим для быстрой проверки логики.

## Формат протокола

Базовый формат кадра:

```text
$cmd,...,*hh\r\n
```

Где:

- `$` — начало кадра.
- `cmd` — тип кадра.
- далее идут код команды и параметры.
- `*hh` — контрольная сумма в hex.
- `\r\n` — окончание кадра.

## Контрольная сумма

Checksum считается как XOR всех ASCII-символов **между** `$` и `*`.

Важно:

- `$` и `*` в расчёт не входят.
- Запятые входят в расчёт.
- Точки входят в расчёт.
- Пробелы входят в расчёт.
- Если в payload есть лишняя запятая или лишний пробел, checksum изменится.

Пример:

```text
payload: cmd,stop,
checksum: 72
frame: $cmd,stop,*72\r\n
```

## Поддерживаемые команды

На текущем этапе в клиенте и эмуляторе реализованы основные команды из документации:

- `cmd,get show,`
- `cmd,get sat,`
- `cmd,sat,<...>,`
- `cmd,get place,`
- `cmd,place,<lon>,<lat>,<heading>,`
- `cmd,reset,`
- `cmd,search,`
- `cmd,stow,`
- `cmd,manual,<adjusting>,<speed>`
- `cmd,dir,<az>,<el>,<pol>,`
- `cmd,stop,`
- `cmd,get rec type,`
- `cmd,set rec,<beacon|dvb>,`
- `cmd,get beacon,`
- `cmd,set beacon,<lo>,<magnification>,`
- `cmd,get dvb,`
- `cmd,set dvb,<lo>,<magnification>,`

## Требования

### Python

Рекомендуется Python 3.

Клиент написан максимально просто и близко по стилю к старому Python 2.x, но в текущем виде тестировался под Python 3.

### Для serial

Если нужен serial-режим, установи `pyserial`:

```bash
pip install pyserial
```

## Быстрый старт

### 1. Запуск эмулятора

```bash
python3 antenna_emulator.py
```

По умолчанию эмулятор слушает:

- host: `127.0.0.1`
- port: `8899`

### 2. Запуск клиента

```bash
python3 antenna_ctl.py tcp 127.0.0.1 8899
```

После этого появится консольное меню команд.

## Если порт занят

Проверить, кто держит порт:

```bash
lsof -i TCP:8899
```

Остановить старый процесс:

```bash
kill <PID>
```

Или запустить эмулятор на другом порту:

```bash
python3 antenna_emulator.py 127.0.0.1 8900
python3 antenna_ctl.py tcp 127.0.0.1 8900
```

## Demo-режим

Для быстрого просмотра логики без клиента:

```bash
python3 antenna_emulator.py demo
```

Этот режим выводит:

- примеры команд,
- рассчитанные checksum,
- тестовые ответы,
- пример ошибки checksum.

## Заметки по протоколу

В процессе сверки с документацией выяснилось несколько важных деталей:

- Последняя запятая перед `*hh` в некоторых командах является частью payload.
- В некоторых командах пробелы тоже являются частью payload и меняют checksum.
- Команда ручного позиционного режима — `dir`, а не `dlr`.
- Команда установки параметров спутника — `sat`, а ответ на неё приходит как `set sat ok`.
- Для speed mode важно буквально повторять формат команды из документации, иначе checksum не совпадёт.

Из-за этого строки команд нельзя автоматически "нормализовать" перед расчётом checksum.
